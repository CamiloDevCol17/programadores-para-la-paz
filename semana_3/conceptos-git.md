**Repositorio:**

Un repositorio es un espacio de almacenamiento donde se guardan los archivos de tu proyecto junto con todo su historial de versiones. Funciona como una base de datos centralizada o local que te permite organizar el código, colaborar con otros y llevar un registro exacto de cada modificación realizada.



**Commit:**

Un commit es básicamente una "fotografía" o un punto de guardado en la línea de tiempo de tu proyecto de software. Se encarga de registrar de forma permanente los cambios que preparaste previamente, permitiéndote tener un historial claro al que puedes volver si cometes algún error en el futuro.



**Push:**

El comando push se utiliza para cargar y enviar los commits que tienes en tu entorno local hacia un repositorio remoto (como GitHub o GitLab). Es el paso final para sincronizar tu trabajo, asegurando que tus cambios estén respaldados en la nube y disponibles para el resto de tu equipo.
